# FaceUnlock Windows 10/11

Реализует `ICredentialProvider` — встраивается в **LogonUI / Secure Desktop**,
работает на настоящем экране блокировки Windows 10/11.

> **Личное удобство, не security-решение.**
> LBPH легко обмануть фотографией. Используйте только как быстрый способ входа,
> на компьютере, которым пользуетесь в основном вы сами (см. раздел
> «Известные ограничения» про безопасность хранения пароля).

---

| Компонент | Где работает | Зачем |
|---|---|---|
| `FaceUnlock.CredentialProvider.dll` | Secure Desktop (LogonUI) | Плитка на экране блокировки, ICredentialProvider |
| `FaceUnlock.Service.exe` | SYSTEM, Session 0 (настоящая служба) | Pipe-сервер, хранение/выдача пароля — должен быть запущен ещё **до** входа в систему |
| `FaceUnlock.RecognizeHelper.exe` | Сессия вошедшего пользователя (её запускает служба через `CreateProcessAsUser`) | Реальный доступ к камере — в Session 0 это невозможно |
| `FaceUnlock.Setup.exe` | Сессия пользователя, трей | Настройка: обучение модели, ввод пароля |

`RecognizeHelper` никогда не видит и не передаёт пароль — он только отвечает
«это то лицо или нет». Пароль расшифровывается и отдаётся в pipe исключительно
внутри `FaceUnlock.Service.exe`.

## Первичная настройка (после установки)

1. В трее появится иконка FaceUnlock (от `FaceUnlock.Setup.exe`)
2. Правая кнопка → **«Настроить лицо и учётные данные...»**
3. **Шаг 1:** Смотрите в камеру, нажимайте «Снять кадр» 10–20 раз
   (слегка поворачивайте голову, меняйте освещение)
4. Нажмите **«Обучить модель»**
5. **Шаг 2:** Введите имя пользователя, домен (или имя ПК для локальных учёток)
   и пароль Windows или Microsoft Account → **«Сохранить»**.
6. Окно настройки также показывает живой статус службы — если там написано
   «служба НЕ отвечает», см. раздел «Диагностика» ниже, прежде чем продолжать
7. Закройте окно настройки

## Использование

1. Заблокируйте ПК (`Win+L`)
2. На экране блокировки появится плитка **«Вход по лицу»** рядом с обычным паролем
3. Кликните по ней — появится статус «Смотрите в камеру...»
4. После 3 подряд совпадений Windows автоматически разблокируется
5. Если не распозналось — кликните на другую плитку и войдите паролем/PIN

> **Важно:** вход по лицу работает для блокировки экрана (`Win+L`) и для
> повторного входа после того, как пользователь хоть раз вошёл в систему
> интерактивно. На самом первом экране входа после **холодной загрузки**
> компьютера (никто ещё не входил) интерактивной пользовательской сессии не
> существует, поэтому службе некуда запускать `RecognizeHelper` — в этом
> случае войдите обычным паролем один раз, дальше `Win+L` будет работать
> по лицу как обычно. Это ограничение самой концепции инструмента, а не баг.

## Хранение данных

Все данные теперь лежат в `%ProgramData%\FaceUnlock\` (не в `%APPDATA%`,
как раньше) — это нужно, чтобы служба, работающая под SYSTEM, могла читать
и расшифровывать их:

- `Data\face_model.xml` — обученная модель LBPH
- `Data\camera_index.txt` — выбранная камера
- `Data\cred_cp.bin`, `Data\username.bin`, `Data\domain.bin` — учётные данные,
  зашифрованные DPAPI (`DataProtectionScope.LocalMachine`)
- `Logs\log.txt` — общий лог Service / Setup / RecognizeHelper

Права на папку `Data\` и `Logs\` ограничены `Install.ps1` через `icacls`:
SYSTEM и Administrators — полный доступ, Users — чтение/запись.

## Диагностика

- **Плитка не появляется на экране блокировки:**
  Проверьте в `regedit`:
  `HKLM\SOFTWARE\Microsoft\Windows\CurrentVersion\Authentication\Credential Providers\{BA0B9B41-...}`
  Должен существовать с именем `FaceUnlock`.

- **«Служба не запущена» на экране блокировки:**
  Откройте `services.msc` → найдите **«FaceUnlock Service»** → статус должен
  быть **«Выполняется»**, тип запуска — **«Автоматически»**. Если служба
  остановлена — запустите её (`Start-Service FaceUnlockService` от администратора)
  и посмотрите `%ProgramData%\FaceUnlock\Logs\log.txt` на предмет ошибок.

- **Иконка в трее показывает «служба НЕ отвечает»:**
  Это означает, что pipe `Global\FaceUnlockRecog` не отвечает — служба либо
  остановлена, либо упала. См. лог и `services.msc`.

- **Лицо не распознаётся:**
  Переснимите кадры в тех же условиях освещения через окно настройки. Порог
  схожести можно понизить в `Shared/FaceRecognizer.cs` (`ConfidenceThreshold`)
  и пересобрать.

- **На экране блокировки лицо не работает только после перезагрузки,
  пока никто не входил:**
  Это ожидаемо — см. раздел «Использование» выше про холодную загрузку.

- **Логи:** `%ProgramData%\FaceUnlock\Logs\log.txt` (общий для всех трёх
  процессов — Service, RecognizeHelper, Setup, с пометкой компонента в начале строки).

## Известные ограничения

- **PIN Windows Hello** не поддерживаются — LogonUI для них использует
  другие пакеты аутентификации (MSA / NGC). Нужна локальная учетка или ввести пароль от Microsoft аккаунта в поле "Пароль учетной записи".
- **Secure Desktop** изолирован от пользовательских приложений — именно поэтому
  нужна архитектура DLL + pipe.
- **Холодная загрузка:** на самом первом экране входа после перезагрузки/включения
  ПК распознавание недоступно — нет ещё ни одной интерактивной сессии, в
  которую служба могла бы запустить `RecognizeHelper`. Первый вход после
  загрузки — паролем, дальше `Win+L` работает по лицу.
- **Быстрый вход (Fast User Switching)** работает, но каждый пользователь
  должен пройти настройку отдельно.
- **Безопасность хранения пароля:** пароль шифруется DPAPI с
  `DataProtectionScope.LocalMachine`, чтобы его могла расшифровать служба
  под SYSTEM. Особенность DPAPI в этом режиме — расшифровать такие данные
  технически может **любой процесс на этой машине** с доступом к файлу,
  а не только SYSTEM. Папка `%ProgramData%\FaceUnlock\Data` ограничена через
  ACL (SYSTEM + Administrators + Users), но на компьютере с несколькими
  локальными учётными записями любой из этих пользователей теоретически
  может прочитать сохранённый пароль. Используйте инструмент на личном ПК
  с одним основным пользователем — как и заявлено в начале этого README,
  это инструмент личного удобства, а не enterprise-решение безопасности.
